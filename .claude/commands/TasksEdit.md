# Improved Structure and User Experience (UX)

## Problem Identified

During the creation and editing flow of a quote, specifically when adding **Services** and **Attributes** to a service, the interface confuses the user.

Currently, when the user fills in the fields for a new Service or Attribute, **there is no visual indication** that the information **has not yet been added** to the list.

For the changes to take effect, the user must press the button:

- **"Add Service"**
- **"Add Attribute"**

However, this action is not obvious, leading the user to believe that the information has already been saved or added automatically.

## Objective

Redesign this flow to make the process more intuitive and reduce the possibility of errors or confusion.

## Considerations

- Make it much clearer that the form represents an element pending addition.

- Give greater prominence to the **"Add Service"** and **"Add Attribute"** buttons.

- Provide visual feedback before and after adding an element.

- Prevent users from abandoning the form believing the information has already been recorded.

- Maintain a consistent experience between creating Services and creating Attributes.

## Expectations

Propose a complete UX/UI improvement, indicating:

1. How the form should look.

2. What visual indicators to add.

3. What changes to make to the buttons.

4. Whether it is advisable to add animations, messages, or visual states.

5. How to reduce the number of clicks without affecting the clarity of the flow.

Prioritize a modern, intuitive, and easy-to-understand interface for users with limited technical experience.
